# 인사이트 및 배운 점 (Insights)

## 🧠 기술적 통찰
- **UX 전략 (Modal vs Page)**: 사용자는 주보와 같은 콘텐츠를 볼 때 즉시 페이지를 이동하기보다 모달로 빠른 확인(Preview)을 한 뒤 필요시 상세로 들어가는 흐름을 선호함.
- **애니메이션 품질**: Framer Motion의 `AnimatePresence`와 `layoutId`(필요시)를 활용하면 단순한 웹이 아닌 앱 같은 매끄러운 느낌을 줄 수 있음.
- **데이터 중앙화의 이점**: `contentData.ts`에 모든 핵심 정보를 몰아넣음으로써 기술적 지식이 부족한 사용자도 데이터 수정만으로 사이트 전체의 정보를 업데이트할 수 있는 구조를 확립함.

## ⚠️ 주의사항 및 팁
- **목사님 성함**: 교회 도메인에서 담임목사님의 정확한 성함 타이핑은 매우 중요하며, 모든 페이지(인사말, 연혁, 설교자 등)에 일관되게 반영되어야 함.
- **복사 기능 UI**: 계좌번호 복사 시 단순 얼럿보다는 아이콘이 체크 표시로 바뀌는 등의 마이크로 인터랙션이 사용자 만족도가 훨씬 높음.
- **Git 작업**: Windows 환경의 PowerShell에서 `&&` 연산자 등의 셸 차이에 주의하고, 푸시 전 항상 사용자 설정 상태를 체크해야 함.
